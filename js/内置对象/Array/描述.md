avaScript 数组是可调整大小的，并且可以包含不同的数据类型。

JavaScript 数组不是关联数组，因此，不能使用任意字符串作为索引访问数组元素，但必须使用非负整数（**或它们各自的字符串形式**）作为索引访问。

>只有 years\['2'] 是一个实际的数组索引。years\['02'] 是一个在数组迭代中不会被访问的任意字符串属性。
console.log(years\["2"] !== years\["02"]); // 这个输出可能是true，我还没试...我试了一下，我的输出是false



> fruits\[5] = "mango";
console.log(fruits\[5]); // 'mango'
console.log(Object.keys(fruits)); // ['0', '1', '2', '5']
console.log(fruits.length); // 6

可以得知增加 **length 会通过添加空槽来扩展数组，而不是创建任何新元素——甚至不会是 undefined。**但是，减少 length 属性会删除元素。较旧的方法诸如 forEach 之类的迭代方法根本不会访问空槽。较新的方法（例如 keys）不会对空槽进行特殊处理，而是将它们视为包含 undefined。

将改变原数组的方法转换为非修改方法的一种简单方式是使用展开语法或 slice() 先**创建一个副本**：
> arr.copyWithin(0, 1, 2); // 改变了 arr
const arr2 = arr.slice().copyWithin(0, 1, 2); // 不改变 arr
const arr3 = \[...arr\].copyWithin(0, 1, 2); // 不改变 arr


尾部可以删除多个元素，发现splice参数为负数时候就是从尾部删除，删除的个数为其绝对值
>const fruits = \["Apple", "Banana", "Strawberry", "Mango", "Cherry"];
const start = -3;
const removedItems = fruits.splice(start);
console.log(fruits);
// \["Apple", "Banana"]
console.log(removedItems);
// \["Strawberry", "Mango", "Cherry"]


如果你想要一个数组的深拷贝，你可以使用 JSON.stringify() 将数组转换成一个 JSON 字符串，然后使用 JSON.parse() 将字符串转换回一个完全独立于原数组的新数组。

对于for-of我们发现item是数组元素，而for-in对应的index则是索引
> for(let item of arr){
    console.log(item)
}
for(let index in arr) {
    console.log(index)
}
