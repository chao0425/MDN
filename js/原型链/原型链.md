JavaScript对象有一条指向原型对象的链。当试图访问对象的属性时，不仅在该对象上查找属性，还会在该对象的原型上查找属性，以及原型的原型，依此类推，直到找到一个名字匹配的属性或到达原型链的末尾。
> js的每个对象都有一条指向原型对象的链，如果访问这个对象的属性，先查找此对象是否具有此属性，若没有，则去原型寻找此属性。直到找到null(null没有原型，所以就是原型链的末尾)

比如如下的代码，c为此对象的原型。若此对象寻找不到某属性，则检查其原型，即检查c
{ a: 1, b: 2, \__proto__: c } 这样的对象字面量中，值c（其必须为null或另一个对象）

在JavaScript中，对象可以以属性的形式添加函数。继承的函数与其他属性一样，包括属性遮蔽（在这种情况下，是一种方法重写的形式）。
> 对象的属性可以是函数，并且从对象继承来的函数可以被遮蔽。即若从此对象中，找到了函数，就不找了，若没找到函数，就检查他的原型。


存在如下语法，child的原型为parent，child继承了parent
> const child = {
  \__proto__: parent,

只能说权威的文档写的案例就是好。。。意思就是如果子类有value了，那么父类的value就被遮蔽了，method虽然是父类的，但是this.value是子类的value。如果子类没有value，那就是父类的value(没找到value就去寻找原型)
>const parent = {
  value: 2,
  method() {
    return this.value + 1;
  },
};
console.log(parent.method()); // 3
// 当调用 parent.method 时，“this”指向了 parent
// child 是一个继承了 parent 的对象
const child = {
  \__proto__: parent,
};
console.log(child.method()); // 3
// 调用 child.method 时，“this”指向了 child。
// 又因为 child 继承的是 parent 的方法，
// 首先在 child 上寻找属性“value”。
// 然而，因为 child 没有名为“value”的自有属性，
// 该属性会在 \[\[Prototype]] 上被找到，即 parent.value。
child.value = 4; // 将 child 上的属性“value”赋值为 4。
// 这会遮蔽 parent 上的“value”属性。
// child 对象现在看起来是这样的：
// { value: 4, \__proto__: { value: 2, method: \[Function] } }
console.log(child.method()); // 5
// 因为 child 现在拥有“value”属性，“this.value”现在表示 child.value

利用原型链，如果某函数需要复用，可以直接在原型里写对实例使用的函数。诶呦，mdn写的太好了。
>// 构造函数
function Box(value) {
  this.value = value;
}
// 使用 Box() 构造函数创建的所有盒子都将具有的属性
Box.prototype.getValue = function () {
  return this.value;
};
const boxes = [new Box(1), new Box(2), new Box(3)];


我们继续看文档怎么说：**类是构造函数的语法糖**,因为Box.prototype引用的对象和所有实例的\[\[Prototype]]是同一个对象，所以我们可以通过改变Box.prototype来改变所有实例的行为。
即__proto__ === Prototype 所以我们修改prototype，即可改变实例的行为
>function Box(value) {
  this.value = value;
}
Box.prototype.getValue = function () {
  return this.value;
};
const box = new Box(1);
// 在创建实例后修改 Box.prototype
Box.prototype.getValue = function () {
  return this.value + 1;
};
box.getValue(); // 2

就是说，我们修改了prototype，在新创建出来的实例引用的是新的原型，而修改前创建出来的是使用的旧的原型。会造成新旧实例的原型分家。即我们有实例a，实例b，若声明实例a后，重写原型，在创建实例b。此后ab就分家了，修改a的原型对b不影响，修改b的原型对a不影响。
